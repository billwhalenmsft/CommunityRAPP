🎤 Demo Talking Points

Key messages for presenting to Ascend stakeholders

💬 "This is your SAP data — we just made it conversational"
Every vendor, contract, GL account, cost center, and DoA level shown here comes from Dataverse tables that mirror your SAP ECC 6.0 master data (LFA1, EKKO, CSKS, T030, T16FS). In production, those same flows read directly from SAP instead of Dataverse — zero re-training needed.
🏭 "We mapped every field back to SAP table columns"
PR Number → BANFN · Vendor → LIFNR · Category → MATKL · GL → SAKTO · Cost Center → KOSTL · Contract → EBELN · Release Status → FRGST. When this goes to production, BAPI_PR_CREATE returns a real BANFN from SAP — no data entry in SAP required.
🛡️ "DoA compliance is enforced automatically"
The agent looks up your T16FS release strategy in real time. A $24,000 PR automatically routes to L2 (John Smith). A $50,000 PR goes to L3 (Susan Lee). $100K+ goes to CFO Office. No one can approve outside their authority — the system blocks it with a specific error message.
📋 "Contract compliance is baked in"
When a vendor + category combination matches an active SAP outline agreement (EKKO), the PR automatically references it. Spend is tracked against the contract maximum value. The agent flags when you're close to the limit or when the contract has expired.
⚡ "From request to PR number in under 3 minutes"
The full 11-step flow — intent detection, vendor lookup, classification, GL derivation, DoA routing, and EBAN write — completes in a single conversation. Compare that to the current ME51N experience.

🔧 Production Upgrade Path (Demo → SAP Live)

Three-step process to switch from Dataverse emulation to live SAP ECC 6.0

① Set Environment Variable
Change ascend_DemoMode from true to false in the Power Platform environment variables panel.
② Configure SAP ERP Connector
Install the on-premises data gateway. Add SAP ERP connector. Set ascend_SAPConnectionString to your SAP hostname, system number, and client.
③ Swap Flow Actions
In each flow, replace Dataverse query actions with SAP ERP connector actions. Comments in each flow JSON label exactly which BAPI/RFC_READ_TABLE to call.

🩺 Troubleshooting

Common issues and fixes

Solution import fails ▶
Ensure the target Power Platform environment has Dataverse enabled (not the default environment for Trial). Check that you're using an unmanaged solution import.
Flows not connected in Copilot Studio topics ▶
In Copilot Studio, open each topic → click the Power Automate action → re-select the flow from the dropdown. This is needed when the flow connection was imported from a different environment.
Sample data import fails (column mapping) ▶
CSV column names must exactly match Dataverse field names (e.g., ascend_lifnr, not LIFNR). Use the Dataverse table column designer to verify field names before importing.
Bot returns generic "I don't understand" ▶
Ensure the bot has been published after import. Unpublished bots use the last published version, which may be empty on first import.
Approval emails not sending ▶
Check that the Office 365 Outlook connection is authorized in Power Automate. Verify ascend_ApprovalNotificationEmail is set. Check Power Automate run history for the specific error.
Flow returns "No records found" for vendors ▶
Confirm sample data was imported into ascend_sapvendor table. The vendor lookup uses a contains() filter on ascend_name1 — try the exact vendor names from the CSV (e.g., "Microsoft", "Dell", "Amazon").
