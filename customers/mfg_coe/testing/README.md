# Customer Test Personas and Scenarios

This directory contains `personas.json` and `scenarios.json` for each customer.
Used by the Customer Persona Agent to simulate realistic customer interactions for testing.

## Structure

Each customer subdirectory contains:
- `personas.json` — Customer persona profiles (name, role, personality, pain points)
- `scenarios.json` — Test scenarios (trigger utterances, expected resolution, D365 data dependencies, success criteria)

## Customers

- `navico/` — Navico marine electronics (warranty, parts, firmware support)
- `otis/` — Otis elevator (service, maintenance, field tech scenarios)
- `zurnelkay/` — Zurn/Elkay (plumbing products, warranty, code compliance)
- `vermeer/` — Vermeer equipment (dealer support, warranty, machine walkaround)
- `carrier/` — Carrier HVAC (case triage, warranty, product support)
- `aes/` — AES scenarios
- `sma4/` — SMA4 sales/manufacturing scenarios
