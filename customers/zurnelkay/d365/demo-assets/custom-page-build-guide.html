Build Steps
1

Create the Custom Page

Power Apps Maker Portal setup
~5 min ▾

1.1 — Open Maker Portal

  1. Go to https://make.powerapps.com
  2. Select the environment: orgecbce8ef (the D365 CS org)
  3. Click Solutions → open or create a solution (e.g., "Zurn Elkay Demo Components")
  4. Click + New → App → Page (Custom Page)
  5. Name: Order Modification Wizard
  6. Choose Start with a blank page

1.2 — Set Page Dimensions

Productivity Pane Width

Custom Pages in the pane render in a narrow side panel. Set Width: 380 and Height: 800.
Go to Settings (⚙️) → Display → update dimensions.

Gallery — galLineItems

Power Fx Filter( 'Sales Order Products', 'Sales Order'.'Sales Order' = varSelectedOrder.'Sales Order' )

Checkbox OnCheck / OnUncheck:

Power Fx // OnCheck Collect(colCancelledLines, ThisItem) // OnUncheck Remove(colCancelledLines, ThisItem)

Running totals footer:

Power Fx "Cancelling " & CountRows(colCancelledLines) & " lines = -$" & Text(Sum(colCancelledLines, 'Extended Amount'), "#,##0.00")

OnVisible — initialize collections:

Power Fx ClearCollect(colCancelledLines, Blank()); ClearCollect(colDateChanges, Blank());

Screen 3: Review & Confirm

Summary of all changes before submitting. Shows delta with red strikethrough on cancellations.

HTML Text control — change summary:

Power Fx "<h3>Lines to Cancel</h3>" & Concat(colCancelledLines, "<div style='color:red;text-decoration:line-through'>Ln " & 'Line Item Number' & " — " & 'Product Description' & " ($" & Text('Extended Amount', "#,##0.00") & ")</div>" ) & "<h3>Delivery Date Changes</h3>" & "<div>New delivery: March 10, 2026 (was March 15)</div>"

Submit button OnSelect:

Power Fx // Cancel the selected lines ForAll(colCancelledLines, Patch('Sales Order Products', ThisRecord, { 'Quantity Cancelled': ThisRecord.Quantity } ) ); // Expedite remaining lines to March 10 ForAll( Filter('Sales Order Products', 'Sales Order'.'Sales Order' = varSelectedOrder.'Sales Order', !('Sales Order Detail' in colCancelledLines.'Sales Order Detail') ), Patch('Sales Order Products', ThisRecord, { 'Ship To: Requested Delivery Date': Date(2026, 3, 10) } ) ); Navigate(scrConfirmation);

Add Data Sources

Connect the Custom Page to Dataverse tables
~3 min ▾

In the Custom Page editor, click Data (cylinder icon) → + Add data → Dataverse, then add each table:

Sales Orders (salesorder) Sales Order Products (salesorderdetail) Accounts (account) Notes (annotation) Cases (incident)
4

Publish & Add to Solution

Save, publish, verify in solution
~2 min ▾
  1. Save the Custom Page (Ctrl+S)
  2. Publish the Custom Page
  3. Verify it appears in your solution (Solutions → your solution → check the Custom Page is listed)
5

Register in the Productivity Pane

The critical step — make it appear in the agent's side panel
~10 min ▾

Option A — Admin Center (Recommended)

  1. Go to https://orgecbce8ef.crm.dynamics.com → Customer Service admin center
  2. Left nav: Workspaces → Agent experience profiles
  3. Open your demo profile (or create "Zurn Elkay Demo Profile")
  4. Under Productivity pane, click Edit → ensure it's Enabled
  5. Click + Add tool → select Custom Page → choose Order Modification Wizard
  6. Set Label: "Order Modification", show on Case entity form, position alongside Copilot & KB Search

Option B — Client API (Programmatic)

If the admin UI doesn't support custom page tools directly, add a Command Bar button on the Case form:

JavaScript function openOrderModificationWizard(formContext) { var pageInput = { pageType: "custom", name: "cr8d7_ordmodwizard_8a1b2", // Replace with your page unique name entityName: "incident", recordId: formContext.data.entity.getId() }; var navigationOptions = { target: 2, // 2 = side dialog width: { value: 400, unit: "px" }, position: 1 // 1 = side pane }; Xrm.Navigation.navigateTo(pageInput, navigationOptions); }

Option C — Agent Script Action

  1. Add an Agent Script step of type "Macro"
  2. Macro action: Open Custom Page (side pane)
  3. The guided walkthrough includes "Process order modification" as a step that opens the wizard
6

End-to-end validation mirroring the phone call demo

Demo Talking Points
"This replaces the Salesforce Screen Flow"
Same guided experience, same side panel position, but built natively on Power Platform with direct Dataverse access.
"No code deployment needed"
Built in the maker portal, deployed through solutions. No Apex triggers, no VisualForce pages, no deployment pipelines.
"Context-aware"
The wizard knows which case the agent is on and auto-filters orders to the customer. In SF, this required custom Lightning components.
"Extensible"
Add approval workflows with Power Automate, connect to FNO for fulfillment, add AI Builder for order anomaly detection — all low-code.
"Same experience as Copilot"
The Order Modification wizard sits right next to Copilot and Smart Assist in the productivity pane. Agents don't leave their workspace.

⏱️ Demo-Day Shortcut (~30 min instead of ~2 hrs)

If you run out of time building the full 4-screen wizard:

  1. Create a single-screen Custom Page with just a gallery showing PO #94820's line items
  2. Add checkboxes for cancel and a submit button
  3. Skip the date picker and review screen

The key demo point is the productivity pane embedding — showing that D365 has the same side-panel guided process capability as Salesforce's Utility Bar flows.

Troubleshooting
Custom Page not showing in pane
Verify it's published and added to the Agent Experience Profile
"No orders found" in gallery
Check Data Source is added; verify Sales Orders table has data
Can't patch Sales Order Products
Ensure the Custom Page has Dataverse connection with sufficient privileges
Productivity pane empty
Enable the pane in Agent Experience Profile → Productivity Pane toggle ON
Custom Page too wide
Set display width to 380px in Custom Page settings
Changes not reflecting
Click refresh on the Sales Order form; Dataverse patches are async
