# Cowork Skills — Public Skill Interface

> **This file is the public gateway to RAPP's M365 Copilot Cowork integration.**
> It tells AI agents (and humans) when and how to target Cowork as a deployment surface.

---

## What is a Cowork Skill?

A Cowork Skill is a `SKILL.md` file (markdown with YAML frontmatter) deployed
to a user's OneDrive at:

```
OneDrive:/Documents/Cowork/skills/{slug}/SKILL.md
```

**M365 Copilot Cowork auto-discovers these at the start of each conversation.**
There is no manifest signing, no Power Automate flow, no Copilot Studio binding
cache, and no Azure infra required. It is by far the simplest deployment target
in the M365 stack.

---

## When to use this target

✅ **Use Cowork Skills when:**
- An SE wants a personal demo surface they fully control (their own OneDrive)
- The agent is primarily about orchestrating M365 data (emails, files, calendar, Teams)
- Zero infrastructure is a hard requirement
- Time-to-demo matters more than scale

❌ **Do NOT use Cowork Skills when:**
- The agent needs custom UI (use `mcp_app` target instead)
- The agent will be shared org-wide with admin governance (use Copilot Studio)
- The agent is heavily transactional (use Azure Functions REST + Foundry)

## How RAPP generates them

```python
from agents.agent_transpiler_agent import AgentTranspilerAgent

t = AgentTranspilerAgent()
result = t.perform(
    action="transpile",
    agent_name="MyAgent",
    target_platform="cowork_skill",
    save_files=True,
)
# Outputs:
#   transpiled/my_agent/cowork_skill/
#       SKILL.md       <- markdown for OneDrive
#       skill.json     <- manifest entry (Cowork-DataLoader compatible)
#       deploy.ps1     <- one-shot Graph upload script
```

## How to deploy

### Manual (one user, fastest)
```powershell
cd transpiled/my_agent/cowork_skill
.\deploy.ps1 -UserEmail seuser@contoso.com
```

### Bulk (across an SE team)
Use the `M365DemoSeeder` agent's `seed_cowork_skills` action with a
`skills.json` manifest at `customers/{customer}/m365_demo_data/skills.json`.

---

## Compatibility

Output is intentionally **schema-compatible with [D365JB/Cowork-DataLoader][cdl]**.
A `skill.json` manifest entry generated by RAPP can be dropped directly into
that tool's `data/skills.json` and deployed by either toolchain.

[cdl]: https://github.com/D365JB/Cowork-DataLoader
